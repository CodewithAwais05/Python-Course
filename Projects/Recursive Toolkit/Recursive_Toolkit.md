# Recursive Toolkit 🔁

A collection of Python programs built to practice and strengthen the concept of **recursion**.
Each program solves a classic problem using a recursive approach (calling a function within itself),
with a clearly defined base case and recursive case.

This folder is part of my ongoing Python practice series, where I implement core programming
concepts through small, focused projects instead of just learning them in theory.

---

## 📌 What's Inside

| # | Program | Description |
|---|---------|-------------|
| 1 | `factorial_of_n.py` | Calculates factorial of a number `n` recursively |
| 2 | `fibonacci_series_upto_n.py` | Prints the Fibonacci series up to `n` terms recursively |
| 3 | `sum_of_digits_of_number.py` | Finds the sum of digits of a number using recursion |
| 4 | `reverse_string.py` | Reverses a string recursively (without using `[::-1]`) |
| 5 | `power_function.py` | Computes `x` raised to the power `n` recursively |
| 6 | `palindrome_string.py` | Checks if a string is a palindrome using recursion |
| 7 | `sum_of_natural_numbers.py` | Calculates the sum of first `n` natural numbers |


---

## 🧠 Concepts Practiced

- Recursive function definition and calling
- Identifying and writing correct **base cases** to avoid infinite recursion
- Breaking a problem into smaller sub-problems (recursive case)
- Tracing recursive call stacks manually to understand execution flow
- Comparing recursive vs iterative approaches for the same problem

## 🖥️ How to Run

Clone this repository and run any file directly using Python:


Each file is self-contained and can be run independently. Sample inputs/outputs are shown
in the code comments or via `input()` prompts where applicable.

## 🎯 Why Recursion?

Recursion is one of the most important problem-solving tools in programming — used heavily
in tree/graph traversal, divide-and-conquer algorithms, backtracking, and dynamic programming.
This folder is my way of building intuition for it through repetition: solving the same *type*
of problem (a function calling itself with a smaller input) across many different contexts.
